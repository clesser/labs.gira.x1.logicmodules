Velux KLF-200 Bausteine Hilfe

Velux KLF-200 Auftraggeber

Logikbausteine Beschreibung Eigenschaften Anwendung Lizenzinformationen

Beschreibung

↑

Dieser Baustein simuliert die Bedieneingänge eines Geräts (Node), einer Gerätegruppe (Group) oder dient dem Abruf einer Szene (Scene). Dabei setzt der Baustein die Steuersignale in Telegramme um, die das Velux KLF 200 Gateway verarbeiten kann.

Für die Telegramme wird eine URL-artige Telegrammsyntax verwendet.

Eingänge

Bezeichnung Porttyp Geltungsbereiche Beschreibung
Zielwert Ganzzahl Node
Group

Empfängt den Zielwert für ein Gerät oder eine Gerätegruppe entsprechend des Eingangswertbereichs.
Zielwert (%) Prozent Node
Group

Empfängt den Zielwert für ein Gerät oder eine Gerätegruppe entsprechend des Eingangswertbereichs.
Min / Max Binär Node
Group

Weist das Gerät oder die Gerätegruppe an, den maximalen (1 = 100%) oder minimalen (0 = 0%) Wert für den Standardparameter zu setzen.

Bei einem Rollladen fährt ein 1-Telegramm den Rollladen vollständig herunter, bei einem 0-Telegramm vollständig herauf.
Stop / Start Binär Node
Group
Scene

Beginnt (1) oder unterbricht (0) die Ausführung einer Szene. Bei Geräten oder Gerätegruppen wir mit (1) der Standardwert eingestellt und mit (0) die Ausführung einer Wertänderung abgebrochen.

Bei einem Rollladen führt eine (1) in der Regel zu keiner Aktion, eine (0) jedoch zum Anhalten des Motors.
Auf / Ab (Lang) Binär Node
Group

Weist das Gerät oder die Gerätegruppe an, den maximalen (1 = 100%) oder minimalen (0 = 0%) Wert für den Standardparameter zu setzen.

Bei einem Rollladen fährt ein 1-Telegramm den Rollladen vollständig herunter, bei einem 0-Telegramm vollständig herauf.
Auf / Ab (Kurz) Binär Node
Group

Die Wertänderungen im Gerät bzw. der Gerätegruppe erfolgen in der definierten Schrittweite (z.B. 10%).

Bei einem Rollladen fährt ein 1-Telegramm den Rollladen um den in der Schrittweite defnierten Wert herunter, bei einem 0-Telegramm - analog dazu - herauf.

Führt das Gerät gerade eine Wertänderung durch (z.B. ein fahrender Rollladen), so wird ein "Auf/Ab (Kurz)" für die "Auf/Ab (Dauer)" als "Stop" interpretiert.

Parameter

Bezeichnung Porttyp Geltungsbereiche Beschreibung
Geltungsbereich Auswahl Node
Group
Scene

Wählt aus, ob der Auftraggeber ein Gerät, eine Gerätegruppe oder eine Szene respräsentieren soll.
Eingabewertbereich Auswahl Node
Group

Legt fest, wie der "Zielwert" eingegeben werden sollen:
Standard (0..51200) erwartet die Rohdaten,
Prozent (0..100) erwartet einen Prozentwert,
Prozent (100..0) erwartet einen Prozentwert und invertiert ihn intern.

Standardwert: Prozent (0..100).
Auf / Ab (Dauer) Zeitspanne Node
Group

Legt die Zeitdauer einer vollständigen Wertänderung (z.B. Auffahrt eines Rollladens) fest. In dieser Zeitspanne werden "Auf / Ab (Kurz)"- Befehle als "Stop"-Befehl ausgeführt. Nach Ablauf dieser Zeit, werden "Auf / Ab (Kurz)"-Befehle wieder als Schritt-Befehle ausgeführt. Weiterhin wird in dieser Zeit das wiederholte Senden eines Wertes unterbunden (es kann jederzeit ein anderer Wert gesendet werden - nur nicht der gleiche. Nach Ablauf der Zeitspanne, wird auch der gleiche Wert wieder als Telegramm geschickt.*

Bei einem Rollladen wird beispielsweise die Zeit für eine vollständige Aufwärtsbewegung (+5-10% Puffer) angegeben.
Auf / Ab (Kurz %) Ganzzahl Node
Group

Legt die Schrittweite für eine Wertänderung über den Eingang "Auf / Ab (Kurz)" in 0..100% fest.
Name oder :Bezeichner Text Node
Group

Name oder Bezeichner des Geräts, der Gerätegruppe oder Szene.

Dem Bezeichner kann ein "/" vorangestellt sein. Es kann zudem der interne Bezeichner (Nummer) genutzt werden, wenn diesem ein ":" (z.B. ":3") vorangestellt wird.
Ausführungsgeschwindigkeit Auswahl Node
Group
Scene

Einige Geräte unterstützen unterschiedliche Geschwindigkeiten für die Wertänderungen. Es kann gewählt werden zwischen:
Standard enstpricht der normalen Ausführungsgeschwindigkeit,
Still reduziert die Geschwindigkeit und
Schnell erhöht sie gegenüber der Normalgeschwindigkeit.

Standardwert: Standard
Absender Auswahl Node
Group
Scene

Definiert den Absender eines Telegramms. Der Standardwert Benutzer - Prio 2 wird für alle menschlichen Interaktionen genutzt. Der Wert Komfort - Prio 2 für alle Automatiken.
Priorität Auswahl Node
Group
Scene

Definiert die Priorität des Telegramms. Der Standardwert Mensch wird für alle menschlichen Interaktionen genutzt. Der Wert Logik / Automatik wird für automatische Steuerungen verwendet.

Anwendung

↑

In der Regel befindet sich ein Gateway-Logikbaustein auf einem Logikblatt, um den herum die verschiedenen Auftraggeber (Rollladen: Küche, Rollladen: Esszimmer, Szene: Urlaub) und die Auwertungsmodule (Rollladen: Küche, Rollladen: Esszimmer) befinden.

Weitere Informationen Beispiele wurden auf der Seite Hinweise zur Einrichtung gesammelt.
