Beschreibung

↑

Dieses Paket enthält Logikbausteine zur Steuerung von Nuki Türschlössern.

Soll nur ein einzelnes Türschloss über die Nuki Bridge gesteuert werden, so kann der Nuki SmartLock verwendet werden.

Sind jedoch mehrere Geräte an die Nuki Bridge angeschlossen, so wird empfohlen, jedes Gerät durch je einen Nuki Auftraggeber und einen Nuki Antwortfilter abzubilden und diese Komponenten mit einer Nuki Bridge zu verbinden, die dann die Kommunikation mit der einzelnen Bridge übernimmt. So lassen sich insbesondere periodisch erfolgende Statusabfragen effizienter und fehlerfreier umsetzen.

Mit Hilfe von Telegrammen können prinzipiell beliebige Geräte angesteuert werden. Die Telegramme können dabei wahlweise auch durch andere Text-Generator-Bausteine erstellt werden.

Logikbausteine

↑
Bezeichnung Beschreibung
Nuki SmartLock

Dieser Baustein ermöglicht das Sperren, Entsperren und Entriegeln eines Türschlosses mittels eines EINZELNEN Nuki SmartLock und liefert Informationen über den Schließzustand, Türsensor und Ladestand.
Nuki Bridge

Dieser Baustein stellt eine Verbindung zu einer Nuki Bridge her und ermöglicht die Verwendung MEHRERER Geräte an einer Nuki Bridge.
Nuki Auftraggeber

Dieser Baustein stellt eine Verbindung zu einer Nuki Bridge her und stellt die Eingangsseite eines Nuki-Geräts dar, mit dem Anforderungen gesendet werden können.
Nuki Antwortfilter

Dieser Baustein stellt eine Verbindung zu einer Nuki Bridge her und stellt die Ausgangsseite eines Nuki-Geräts dar, mit dem die Statusinformationen ausgelesen werden können.

Die drei unteren Bausteine unterhalten sich untereinander mit Hilfe von Telegrammen, die an eine URL angelehnt sind und somit bei Bedarf auch manuell erzeugt bzw. abgerufen werden können.

Changelog

↑
Version Änderungen
1.0.1

Geräte-ID kann nun alphanumerische Zeichen enthalten.

Da neuere Nuki-Geräte-IDs nicht mehr ausschließlich aus Ziffern bestehen, wurde das Datenfeld von "Zahl" auf "Text" geändert. Der Baustein muss gegebenenfalls einmal vom Logikblatt entfernt und neu hinzugefügrt werden, damit die Änderung wirksam wird.
1.0.0

Neue interne Architekur des bzw. der Nuki-Bausteine.
ACHTUNG: Aufgrund der veränderten Architektur und damit verbundenen Ein-/Ausgänge der Nuki-Logikbausteine müssen alle verwendeten Bausteine vor der Aktualisierung aus den Logikblättern entfernt und nach der Aktualisierung neu hinzugefügt werden!

Die neue verteilte Architekur ermöglicht es sowohl einzelne Nuki Geräte, als auch mehrere Geräte an einer Nuki Bridge zu betreiben.
0.9.5

Token wird nicht mehr gehashed übertragen (von Nuki als "nicht mehr empfohlen" bezeichnet).

Ab dieser Version wird das Token im Klartext übertragen!
0.9.0 Erste veröffentlichte Version

Die aktuelle Version des Logikpakets ist im KNX User Forum und im AppStore für den Gira X1 verfügbar. Der Source Code liegt auf GitHub.
